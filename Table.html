Tables

1. A table is defined using a 'table' element. This contains a number of table cells ('td' which stands for table data).
These are organized into table rows ('tr'). The HTML code for a table is always based on rows, never on columns.

2. Table cells which act as column headers or row headers should be using the 'th' (table header) element.

3. Table cells can be merged using the colspan and rowspan attributes.

4. Tables can be broken into sections using the following elements:

  • Table Header

  • Table Body

  • Table Footer


    • 5. A caption can be added to the table using the 'caption' element. You can use 'col' and 'colgroup' to define table columns for styling, however, there are a number of limitations with this practise.

    About Table-Based Layout

    It was common in the early days of the web to use tables as a layout device. Before the advent of modern standards-based browsers, this was the easiest way to make sure that page elements were arranged properly on the screen.

    This design pattern is now considered very bad. It is bad for the user experience, bad for SEO, and bad for developers who have to maintain pages.

    You should not use table-based layout under any circumstances. Instead, check out our CSS Tutorials to start learning about modern web site layout.

    However, that doesn't mean you should avoid tables — tables should be used whenever you need to present information in a tabular format.
